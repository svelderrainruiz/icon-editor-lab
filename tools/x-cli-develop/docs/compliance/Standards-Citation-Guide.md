# Standards Citation Guide (Repo‑local)

**How to cite (in any doc or PR):**

-   Use **bracket style**: **[Standard — Clause title]**. Examples:
    -   PR gate rationale → **[29119‑2 — Test monitoring & control]**
    -   Release gate evidence → **[12207 — Transition/Verification]**
    -   Baseline/tag policy → **[10007 — Configuration identification]**
-   Prefer **titles over numbers** (edition numbering varies). Keep direct quotes ≤ **25 words**; otherwise **paraphrase + citation**.
-   Pin editions in _Standards‑Edition‑Appendix.md_. If an edition changes, update the Appendix, not every doc.

**Do/Don’t**

-   ✅ “We enforce thresholds at PR as exit criteria **[29119‑2 — Test monitoring & control]**.”
-   ❌ Long block quotes or uncited claims.
-   ✅ Map each doc to **15289** info‑item type.
-   ✅ For CM topics, pair **10007** (process) with **828** (plan content).

**Common anchors**

-   Test gates → **[29119‑2 — Monitoring & control]**
-   Test records → **[29119‑3 — Test report]**
-   Requirements/RTM → **[29148 — Requirements information items]**
-   Architecture packet → **[42010 — Viewpoints, views, correspondences]**
-   CM baselines/controls → **[10007 — Identification; Change control; Status accounting; Audit]**
-   CM plan scope → **[IEEE 828 — SCMP content]**
