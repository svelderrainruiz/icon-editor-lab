# Render-VIHistoryReport.ps1

**Path:** `tools/Render-VIHistoryReport.ps1`

## Synopsis
Transforms a `pr-vi-history-summary@v1` manifest into Markdown (and optional HTML) so VI history runs can be shared with stakeholders.

## Description
- Accepts the manifest generated by `Invoke-PRVIHistory.ps1` plus optional context JSON (lineage metadata) and produces human-friendly tables summarizing totals, lineage, attribute coverage, and per-target results.
- When `-EmitHtml` or explicit `-HtmlPath`/`-MarkdownPath` are provided, writes both formats to disk; otherwise, defaults to `<OutputDir>/history-report.md/html`.
- Integrates with GitHub Actions via `-GitHubOutputPath` and `-StepSummaryPath`, appending links to published reports and totals for workflow summaries.

### Parameters
| Name | Type | Default | Notes |
| --- | --- | --- | --- |
| `ManifestPath` | string (required) | - | Path to the `pr-vi-history-summary@v1` JSON file. |
| `HistoryContextPath` | string | - | Optional JSON describing lineage/branch buckets. |
| `OutputDir` | string | `tests/results/pr-vi-history/<timestamp>` | Base folder for generated artifacts. |
| `MarkdownPath` / `HtmlPath` | string | auto | Explicit output paths (when omitted they land under `OutputDir`). |
| `EmitHtml` | switch | Off | Force HTML emission even if `HtmlPath` isnâ€™t set. |
| `GitHubOutputPath` | string | - | When set, appends `markdown_path`/`html_path` entries for GHA steps. |
| `StepSummaryPath` | string | - | Writes a compact summary to the GitHub step summary file. |

## Outputs
- Markdown + HTML run reports (lineage tables, attribute coverage, per-target status, delta summaries).
- Optional GitHub output entries so downstream steps can reference the generated report.
